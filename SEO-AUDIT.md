## ❌ Remaining TODO

### 1. ❌ No legal or trust pages exist anywhere

**Files:** `app/` (entire directory — confirmed absent)

There are **zero** routes for `/privacy`, `/terms`, `/dmca`, `/contact`, `/about`,
or `/cookie-policy`. The footer (`layout.tsx`) links only to `/home`, `/browse`,
`/search`.

**Why it matters:** Google's E-E-A-T system looks hard for legal documentation, a
contact path, and an identifiable operator — especially on a streaming site. DMCA
compliance also legally requires a designated agent + takedown procedure.

**Fix:** Create real, indexable routes and link them from the footer:
- `app/privacy/page.tsx` — Privacy Policy (must disclose GA tracking)
- `app/terms/page.tsx` — Terms of Service
- `app/dmca/page.tsx` — DMCA notice with takedown email + procedure
- `app/contact/page.tsx` — Contact info
- `app/about/page.tsx` — About the service

---

### 2. ❌ Obfuscated popunder ad from a low-tier network

**Files:** `components/Popunder.tsx`, `app/layout.tsx`

The popunder injects an obfuscated IIFE from `antiadblocksystems.com` and a
Cloudfront domain disguised as jQuery libraries. Three compounding problems:
intrusive interstitial penalty, ad-network reputation, and obfuscation red flag.

**Fix:** Remove `<Popunder />` from `layout.tsx`. If ads are needed, use a
reputable network with in-content placements.

---

### 8. ❌ Footer has no contact, entity, or takedown path

**Files:** `app/layout.tsx`

The footer is two deflective sentences plus four nav links. No contact email, no
DMCA agent, no physical address, no operator entity name.

**Fix:** Add operator entity name, contact email, and links to the legal pages
from Finding 1.

### 9. ❌ Organization JSON-LD missing contactPoint, address, sameAs

**File:** `app/layout.tsx`

The Organization schema has only `name`, `url`, `logo`, `description`. Missing
the fields Google uses to validate a real entity.

**Fix:** Add `contactPoint`, `address`, and `sameAs` (only real profiles).

## ✅ COMPLETED — What Was Fixed This Session

### 3. ✅ Duplicate content — slug redirect

**Files:** `app/movie/[slug]/page.tsx`, `app/tv/[slug]/page.tsx`

Both detail pages now compute the canonical path via `buildHref(type, item)` and
308-redirect any non-matching slug. `generateMetadata` also uses the canonical
path, so the `<link rel="canonical">` is always correct. All breadcrumbs, JSON-LD
URLs, and internal links now point to the one true URL.

### 4. ✅ `/api/` proxy blocked from crawlers

**Files:** `app/robots.ts`, `app/api/tmdb/[...path]/route.ts`

`robots.txt` now disallows `/api/`, and the API route sends `X-Robots-Tag: noindex`
as defense-in-depth.

### 5. ✅ Browse pagination crawlable

**Files:** `components/Browse.tsx`, `app/globals.css`

Prev/Next are real `<a href>` links with `rel="prev"`/`rel="next"`, built from a
shared `buildBrowseUrl()` helper. Clicks use `preventDefault` for smooth
client-side UX.

### 6. ✅ Zero-result browse pages noindexed + empty state

**Files:** `app/browse/page.tsx`, `components/Browse.tsx`, `app/globals.css`

`generateMetadata` is now async — it fetches the discover call (deduplicated by
Next.js with the page component) and sets `noindex,follow` when results are empty.
The client renders a helpful empty state with a "browse all titles" link, and
pagination is hidden when there are no results.

### 7. ✅ `/home` H1 fixed

**Files:** `components/Banner.tsx`, `app/home/page.tsx`, `app/globals.css`

Banner title demoted from `<h1>` to `<h2>`. Added `<h1 className="sr-only">`
with a stable, keyword-targeted page heading. All 6 CSS selectors updated from
`.banner-content h1` to `.banner-content h2`.

### 10. ✅ Meta keywords deleted

**File:** `app/layout.tsx`

The keyword-stuffed `keywords` field is removed — eliminates a spam signal.

### 11. ✅ Detail poster fetchPriority

**File:** `components/DetailView.tsx`

Added `fetchPriority="high"` to the LCP poster `<img>` on detail pages.

### 12. ✅ Clock re-renders eliminated

**Files:** `components/Navbar.tsx`, `components/Banner.tsx`

Both clocks now write directly to a ref'd DOM node (`el.textContent = ...`)
instead of calling `setState`. Zero React re-renders per second.

### 14. ✅ Faceted browse URLs noindexed

**File:** `app/browse/page.tsx`

Year/country filter combos now return `noindex,follow`. Only `type` × `genre`
category pages remain indexable.

### 15. ✅ Sitemap URL fixed

**File:** `app/sitemap.ts`

Sitemap entry changed from `/browse` to `/browse?type=movie` to match the
canonical.

### 16. ✅ Unicode slugify

**File:** `lib/slug.ts`

`slugify()` now uses NFD decomposition (for Latin diacritics) + `\p{L}`/`\p{N}`
Unicode property escapes (to keep CJK/Korean/ Japanese) + NFC recombination (to
recombine Hangul jamo). Korean dramas and anime get keyword-rich URLs like
`/tv/오징어게임-94605` instead of `/tv/94605`.

### 17. ✅ Thin detail pages enriched

**Files:** `lib/detail.ts`, `components/DetailView.tsx`

New `synopsisOf()` helper generates 2–3 title-specific sentences from cast,
genres, runtime, and languages when TMDB has no overview. Wired into the meta
description, JSON-LD `description`, and visible `<p>` — all three stay in sync.

### 18. ✅ GA optimized

**File:** `app/layout.tsx`

Both gtag scripts changed to `strategy="lazyOnload"`. Added preconnect to
`www.googletagmanager.com`.

### 19. ✅ PWA manifest complete

**Files:** `public/manifest.json`, `public/icon-512.png`, `public/icon-512-maskable.png`

Generated 512×512 + maskable icons (80% safe zone on black). Also fixed
`icon-192.png` which was 192×128 (mismatched with its declared 192×192).
Manifest now has 3 icons with `purpose: "any"` / `"maskable"`.
