# SEO & Indexability Audit — JustFlixMovies

> Generated by a 5-agent parallel audit covering Technical SEO, Content Quality,
> Performance/Core Web Vitals, Trust/Legal/Shadowban Risk, and Site Architecture.

---

## Executive Summary

The technical SEO foundation is **surprisingly strong** — canonicals, JSON-LD
schema, sitemap, meta tags, and robots directives are all deliberately and
correctly implemented. However, there are a handful of **critical gaps** that
pose real indexation and penalty risks, especially for a free streaming site
(which Google scrutinizes heavily).

### Fix these first (by impact)

| Priority | Issue | Effort |
|----------|-------|--------|
| 🔴 1 | No legal/trust pages (Privacy, Terms, DMCA, Contact, About) | Medium |

## 🔴 CRITICAL — Fix Immediately

### 1. No legal or trust pages exist anywhere

**Files:** `app/` (entire directory — confirmed absent)

There are **zero** routes for `/privacy`, `/terms`, `/dmca`, `/contact`, `/about`,
or `/cookie-policy`. The footer (`layout.tsx:187-192`) links only to `/home`,
`/browse`, `/search`.

**Why it matters:** Google's E-E-A-T (Experience, Expertise, Authoritativeness,
Trustworthiness) system looks hard for legal documentation, a contact path, and
an identifiable operator — especially on a streaming site. A site offering
copyrighted movie streams with no legal pages, no contact, and no DMCA process
is the textbook profile that gets flagged as untrustworthy and demoted.

DMCA compliance also **legally requires** a designated agent + takedown procedure
for safe-harbor protection. You currently have neither.

**Fix:** Create real, indexable routes and link them from the footer:
- `app/privacy/page.tsx` — Privacy Policy (must disclose GA tracking)
- `app/terms/page.tsx` — Terms of Service
- `app/dmca/page.tsx` — DMCA notice with takedown email + procedure
- `app/contact/page.tsx` — Contact info
- `app/about/page.tsx` — About the service

### 2. Obfuscated popunder ad from a low-tier network

**Files:** `components/Popunder.tsx:14`, `app/layout.tsx:159`

The popunder injects an obfuscated IIFE from `antiadblocksystems.com` and a
Cloudfront domain disguised as jQuery libraries (`ljquery.simplePagination`,
`kjquery.terminal`). It fires a popunder on user interaction.

**Why it matters — three compounding problems:**
1. **Intrusive interstitial penalty** — Google explicitly penalizes ads that open
   new tabs/windows on click. This is the textbook trigger.
2. **Ad-network reputation** — Domains from "antiadblocksystems" plus obfuscated
   loaders are the signature of networks that serve malware/redirects/scam ads.
   Google Safe Browsing flags sites associated with these.
3. **Obfuscation is a quality red flag** — Rater guidelines treat deliberately
   obfuscated scripts as suspicious.

**Fix:** Remove `<Popunder />` from `layout.tsx`. If ads are needed, use a
reputable network (AdSense, Ezoic) with in-content placements — never a popunder,
never obfuscated.

### 3. Duplicate content — slug is decorative, canonical self-references the request

**Files:** `lib/slug.ts:28-31`, `app/movie/[slug]/page.tsx:24`,
`app/tv/[slug]/page.tsx:21`, `lib/detail.ts:37,58`

`idFromSlug()` extracts only the trailing numeric ID and ignores the slug text.
The canonical is then built from the **incoming** slug, not the correct one:

```
/movie/avatar-83533      → canonical: /movie/avatar-83533
/movie/wrong-title-83533 → canonical: /movie/wrong-title-83533  (SAME content)
/movie/83533             → canonical: /movie/83533              (SAME content)
```

Each variant serves identical content but emits a **different** self-referencing
canonical. Anyone can generate millions of URLs by varying the slug text. There
is no middleware and no redirect to enforce a canonical slug.

**Fix:** In both detail pages, compute the canonical slug from the fetched item
and 301/308-redirect non-canonical slugs:

```ts
import { redirect } from 'next/navigation';
import { buildHref } from '@/lib/slug';

const canonicalPath = buildHref('movie', item);
if (canonicalPath !== `/movie/${params.slug}`) {
  redirect(canonicalPath); // permanent — collapses all variants
}
```

Then pass `canonicalPath` to `buildDetailMetadata` and `DetailView`.

### 4. `/api/` catch-all proxy is fully crawlable

**Files:** `app/robots.ts:7` (`allow: '/'`, no disallow),
`app/api/tmdb/[...path]/route.ts` (no `X-Robots-Tag`)

`robots.txt` allows everything. The catch-all API route proxies **any** TMDB
endpoint with the server-side API key. Googlebot can discover and crawl URLs
like `/api/tmdb/discover/movie?...` — raw JSON that wastes crawl budget and
Worker invocations.

**Fix (both):**

```ts
// app/robots.ts — disallow the API
rules: [{ userAgent: '*', allow: '/', disallow: ['/api/'] }],
```

```ts
// app/api/tmdb/[...path]/route.ts — add noindex header
headers: { 'x-robots-tag': 'noindex' }
```

## 🟠 HIGH — Fix Soon

### 5. Browse pagination is uncrawlable

**Files:** `components/Browse.tsx:180-190`, `app/browse/page.tsx:74-80`

Prev/Next are `<button onClick>` elements — not `<a href>`. Googlebot cannot
follow them. Pages 2+ of every browse filter are orphaned: they exist as URLs
(`?page=2`) but have no crawlable links pointing to them. Deep catalog titles
beyond page 1 are invisible to crawlers.

**Fix:** Render pagination as real `<Link>` anchors:

```tsx
<Link href={`/browse?${new URLSearchParams({ ...filters, page: String(p + 1) })}`}>
  Next
</Link>
```

### 6. Zero-result browse pages are indexable with no content

**Files:** `app/browse/page.tsx:57` (`robots: { index: true }`),
`components/Browse.tsx:172-190` (no empty state)

When a niche filter combo returns nothing, the page renders just an `<h1>` and
empty grid with "Page 1 of 0" — thin content that's still marked `index: true`.

**Fix:**
- Set `robots: { index: false, follow: true }` when `items.length === 0`
- Add a helpful empty state ("No titles match — try widening your filters")

### 7. `/home` H1 is a rotating movie title, not a page heading

**Files:** `components/Banner.tsx:71`, `app/home/page.tsx:60-75`

The page's sole `<h1>` is whatever movie the banner slideshow happens to show.
It changes every 5 seconds and doesn't match the page's target keyword
("Trending Movies & TV Shows to Watch Now").

**Fix:** Demote the banner title to `<h2>` and add a page-level `<h1>` like
"Trending Movies & TV Shows to Watch Free Online".

### 8. Footer has no contact, entity, or takedown path

**Files:** `app/layout.tsx:178-195`

The footer is two deflective sentences plus four nav links. No contact email, no
DMCA agent, no physical address, no operator entity name. The disclaimer ("Legal

### 9. Organization JSON-LD missing contactPoint, address, sameAs

**File:** `app/layout.tsx:119-127`

The Organization schema has only `name`, `url`, `logo`, `description`. Missing
the fields Google uses to validate a real entity: `contactPoint`, `address`,
`sameAs` (social profiles). This is a negative trust signal for a streaming site.

**Fix:** Add `contactPoint`, `address`, and `sameAs` (only real profiles — don't
fabricate them).

### 10. Keyword-stuffed meta keywords (spam signal)

**File:** `app/layout.tsx:54-56`

Google ignores the `keywords` meta for ranking, but quality raters and spam
classifiers read obvious keyword stuffing as a spam signal. The current list has
10 near-duplicate variations.

**Fix:** Delete the `keywords` field entirely.

### 11. Detail poster missing `fetchPriority="high"` (LCP on highest-traffic pages)

**File:** `components/DetailView.tsx:88-94`

On `/movie/[slug]` and `/tv/[slug]`, the poster is the LCP element. It has no
`fetchPriority` and no `<link rel="preload">`. Unlike `/home` (which got a proper
preload), the site's highest-traffic pages get no LCP optimization.

**Fix:** Add `fetchPriority="high"` to the poster `<img>`:

```tsx
<img className="detail-seo-poster" src={poster} alt={`${title} poster`}
  width={300} height={450} fetchPriority="high" />
```

---

### 12. Two 1-second `setInterval` re-renders (INP cost)

**Files:** `components/Navbar.tsx:45-50`, `components/Banner.tsx:40-45`

The Navbar (mounted globally on every page) updates React state every second for
a decorative clock. On `/home`, Banner does the same — two concurrent timers.
Each tick triggers React render + reconciliation + DOM write, inflating INP.

**Fix:** Move the clock out of React state — write directly to a ref'd DOM node:

```tsx
const clockRef = useRef<HTMLSpanElement>(null);
useEffect(() => {
  const tick = () => {
    if (clockRef.current)
      clockRef.current.textContent = new Date().toLocaleTimeString('en-GB', { hour12: false });
  };
  tick();
  const id = setInterval(tick, 1000);
  return () => clearInterval(id);
}, []);
// JSX: <b ref={clockRef}>--:--:--</b>
```

### 13. Boot overlay hides content for ~0.55s on first load

**File:** `components/SystemChrome.tsx:109-113`

The full-screen "INITIALISING FREE_STREAM_TERMINAL" overlay covers the page
during the LCP window. Depending on paint timing, the overlay text can itself
become the reported LCP element, then get replaced — inflating measured LCP.

**Fix:** Remove it, or shorten to ~150ms.

### 14. ~34,000 faceted browse URLs allowed to index but unreachable by crawl

**Files:** `lib/browse.ts:8-26`, `app/browse/page.tsx:35-42`

The browse canonical includes `type` × `genre` × `year` × `country` =
~34,000 possible indexable URLs. But filters are `<select>` elements (not `<a>`
links), so crawlers can't discover them. If they ever get indexed via external
links, they consume crawl budget with thin, overlapping content.

**Fix:** Noindex year/country filter combos:

```ts
robots: (f.year || f.country)
  ? { index: false, follow: true }
  : { index: true, follow: true },
```

### 15. Sitemap URL `/browse` ≠ canonical `/browse?type=movie`

**Files:** `app/sitemap.ts:55`, `app/browse/page.tsx:37`

The sitemap submits `/browse`, but that page canonicalizes to `/browse?type=movie`.
Google must follow the canonical signal — a wasted crawl round-trip.

**Fix:** Change the sitemap entry to `${SITE_URL}/browse?type=movie`.

### 16. Non-ASCII titles produce ID-only URLs (no keywords for K-drama/anime)

**File:** `lib/slug.ts:6-11`

`slugify()` strips all non-Latin characters (`[^\w\s-]`). Korean dramas and anime
— core content verticals — get slugs like `/tv/94605` with zero keywords.

**Fix:** Transliterate non-ASCII titles to Latin, or URL-encode the original
title so it's at least present in the URL.

---

### 17. Detail pages with no TMDB overview are thin

**File:** `lib/detail.ts:38`, `components/DetailView.tsx:101`

Titles without a TMDB overview fall back to one boilerplate sentence: *"Watch
{title} online free in HD on JustFlixMovies."* The entire body is h1 + meta line
+ that sentence. Can trip thin-content heuristics.

**Fix:** Enrich the body with cast list, genres, runtime, release date, country,
and a "More Like This" intro so even overview-less titles have substantial text.

### 18. GA could use `lazyOnload` + preconnect

**Files:** `app/layout.tsx:146-158`

Both gtag scripts use `afterInteractive`. GA4 has no immediate functional need.

**Fix:**
- Change to `strategy="lazyOnload"` on both `<Script>` tags
- Add `<link rel="preconnect" href="https://www.googletagmanager.com" />`

### 19. PWA manifest missing 512×512 and maskable icons

**File:** `public/manifest.json:9-15`

Only a 192×192 icon exists. Chrome requires 192+512 for installability. No
`purpose: "maskable"` means Android crops the icon unpredictably.

**Fix:** Add `icon-512.png` and a maskable variant; update the manifest.

## ✅ What's Already Done Well (Don't Touch)

These were verified correct — no action needed:

- **Canonical strategy** — correct on every page type (landing, home, browse,
  search, detail). Browse correctly excludes `sort`/`page` from canonical.
- **Robots directives** — global `index,follow`; search results and 404
  correctly `noindex,follow`. No accidental noindex on indexable pages.
- **JSON-LD schema** — comprehensive: WebSite, Organization, FAQPage, ItemList,
  BreadcrumbList, Movie/TVSeries, VideoObject trailer, AggregateRating.
  All mirror visible on-page content (Google requirement).
- **Sitemap** — dynamic, covers ~800 detail URLs + category pages, dedupes,
  handles TMDB date edge cases, escapes XML correctly.
- **Self-hosted fonts** — `next/font` eliminates render-blocking font requests.
- **Inline SVG icons** — Font Awesome CDN eliminated; zero icon HTTP requests.
- **Poster cards are real `<a href>`** — crawlable without JS (progressive
  enhancement).
- **Image dimensions** — all `<img>` have explicit width/height (CLS controlled).
- **LCP preload on `/home`** — banner backdrop preloaded with `fetchPriority:high`.
- **No cloaking or sneaky redirects** — SSR HTML matches what users see.
- **Banned IDs** — filtered from listings, search, recommendations, JSON-LD,
  and sitemap; detail URLs 404.
- **SEO copy is visible** — not `display:none`, no hidden-text risk.
